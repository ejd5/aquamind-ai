Comment ça marche ?

Le toolkit scripts/i18n/toolkit.py utilise notre LLM GLM-5.2 déjà installé via z-ai-web-dev-sdk. Quand vous ajoutez une nouvelle clé en français, le toolkit :

  1. Détecte les clés manquantes dans les 6 autres langues
  2. Envoie le texte français à GLM pour traduction
  3. Préserve les placeholders {param} et les termes techniques
  4. Met à jour automatiquement les 6 fichiers de locale
  5. Valide la couverture et la qualité

Commandes disponibles

1. Voir l'état des traductions

python3 scripts/i18n/toolkit.py status

Affiche le nombre de clés par langue, les clés manquantes, et vérifie les placeholders.

2. Traduire les clés manquantes

python3 scripts/i18n/toolkit.py translate

Détecte toutes les clés manquantes et les traduit automatiquement avec GLM vers les 6 langues.

3. Traduire une clé spécifique

python3 scripts/i18n/toolkit.py translate-key "common.maNouvelleCle" "Texte en français à traduire"

Traduit une seule clé dans les 6 langues et sauvegarde dans les fichiers.

4. Valider la couverture

python3 scripts/i18n/toolkit.py validate

Vérifie : couverture des clés, valeurs vides, placeholders cassés.

Workflow quotidien

Quand vous ajoutez une nouvelle fonctionnalité

  1. Ajoutez la clé dans src/i18n/locales/fr.json
  2. Utilisez t("cle") dans votre composant
  3. Exécutez : python3 scripts/i18n/toolkit.py translate
  4. GLM traduit automatiquement vers EN, ES, DE, IT, PT, NL
  5. Vérifiez : python3 scripts/i18n/toolkit.py status
  6. Committez — le pre-commit hook vérifie qu'aucun texte codé en dur n'est ajouté

Règles de traduction GLM

Le prompt GLM inclut ces règles :

  • ✅ Préserve les placeholders {param} exactement
  • ✅ Garde les noms de marque : AQWELIA, TAC, CYA, pH
  • ✅ Garde les termes techniques : skimmer, backwash, ppm, mg/L
  • ✅ Traduit naturellement (pas mot à mot)
  • ✅ Retourne uniquement la traduction
